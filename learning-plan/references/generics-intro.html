1. ジェネリクスとは何か？

ジェネリクスとは、型をパラメータとして受け取る仕組みです。 関数・クラス・インターフェースを定義するとき、具体的な型を後から指定できるようにすることで、 型安全性を保ちながらコードを再利用できます。

💡
一言でいうと： 「どんな型でも扱えるが、使うときには型が決まる」仕組みです。
Array<string> や Promise<number> はジェネリクスを使った型の代表例です。

ジェネリクスがない世界

any を使う（型が消える）
function identity(value: any): any {
  return value;
}

const result = identity("hello");
// result の型は any → 補完も型チェックもなし
result.toFixed(2); // 実行時にクラッシュ
ジェネリクスを使う
function identity<T>(value: T): T {
  return value;
}

const result = identity("hello");
// result の型は string → 補完・型チェックが働く
result.toUpperCase(); // OK
result.toFixed(2);    // コンパイルエラー

2. なぜ必要なのか ─ 問題提起

「同じロジックだが、扱う型が違う」という状況は頻繁に発生します。

問題：型ごとに同じ関数を書き直す羽目になる

without-generics.ts ─ コードが重複する
// 数値の配列から最初の要素を返す
function firstNumber(arr: number[]): number | undefined {
  return arr[0];
}

// 文字列の配列から最初の要素を返す（ロジックは同じ！）
function firstString(arr: string[]): string | undefined {
  return arr[0];
}

// User の配列から最初の要素を返す（またロジックは同じ…）
function firstUser(arr: User[]): User | undefined {
  return arr[0];
}

解決：ジェネリクスで1つにまとめる

with-generics.ts
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}

first([1, 2, 3]);          // 型: number | undefined
first(["a", "b"]);        // 型: string | undefined
first([{ id: 1 }, { id: 2 }]); // 型: { id: number } | undefined
✅
ジェネリクスを使うと DRY（Don't Repeat Yourself） を守りながら、 型安全性を犠牲にしないコードが書けます。

3. 基本構文の解剖

ジェネリクスは < >（山括弧）の中に型パラメータを書きます。慣習として T、U、K、V などが使われます。

function wrap<T>(value: T): { data: T } {
  return { data: value };
}
関数名
型パラメータ宣言
型パラメータの使用（引数）
型パラメータの使用（戻り値）

型パラメータの命名慣習

名前典型的な用途例
T 汎用の型（Type） Array<T>, Promise<T>
K, V キー（Key）と値（Value） Map<K, V>, Record<K, V>
E 要素（Element） Set<E>
R 戻り値（Return type） (...args: any[]) => R

型推論が働くケース

型推論（引数から T が自動決定される）
function wrap<T>(value: T): { data: T } {
  return { data: value };
}

// 明示的に型を指定する場合
wrap<string>("hello");

// 型推論に任せる（推奨）─ 引数の型から T = string が決まる
wrap("hello");  // T は string と推論される
wrap(42);       // T は number と推論される
wrap({ id: 1 }); // T は { id: number } と推論される

4. ジェネリック関数

例1：配列操作ユーティリティ

array-utils.ts
// 配列をシャッフルして返す
function shuffle<T>(arr: T[]): T[] {
  return [...arr].sort(() => Math.random() - 0.5);
}

// 配列から重複を除く
function unique<T>(arr: T[]): T[] {
  return [...new Set(arr)];
}

// 配列をグループ化する
function groupBy<T, K extends string | number>(
  arr: T[],
  keyFn: (item: T) => K
): Record<K, T[]> {
  return arr.reduce((acc, item) => {
    const key = keyFn(item);
    acc[key] = [...(acc[key] ?? []), item];
    return acc;
  }, {} as Record<K, T[]>);
}

// --- 使い方 ---
const users = [
  { name: "Alice", role: "admin" },
  { name: "Bob",   role: "user"  },
  { name: "Carol", role: "admin" },
];

const grouped = groupBy(users, u => u.role);
// { admin: [{name:"Alice",...}, {name:"Carol",...}], user: [{name:"Bob",...}] }

例2：API レスポンスのラッパー関数

api-helpers.ts
type ApiResponse<T> = {
  data: T;
  status: number;
  message: string;
};

async function fetchJson<T>(url: string): Promise<ApiResponse<T>> {
  const res = await fetch(url);
  const data: T = await res.json();
  return { data, status: res.status, message: "ok" };
}

// --- 使い方 ---
type Product = { id: number; name: string; price: number };

const result = await fetchJson<Product>("/api/products/1");
// result.data は Product 型として補完される
console.log(result.data.price); // OK
console.log(result.data.stock); // コンパイルエラー（Product に stock はない）

5. ジェネリッククラス

クラスにも型パラメータを付けることができます。データ構造やリポジトリパターンで特に活躍します。

例1：スタック（Stack）データ構造

stack.ts
class Stack<T> {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  isEmpty(): boolean {
    return this.items.length === 0;
  }
}

// --- 使い方 ---
const numStack = new Stack<number>();
numStack.push(1);
numStack.push(2);
numStack.push("3"); // コンパイルエラー！ string は入れられない

const strStack = new Stack<string>();
strStack.push("hello");
strStack.push("world");
strStack.pop(); // "world"（型は string）

例2：汎用リポジトリクラス（実務パターン）

repository.ts ─ 在庫管理システムのイメージ
interface Entity {
  id: number;
}

class Repository<T extends Entity> {
  private store: Map<number, T> = new Map();

  save(entity: T): T {
    this.store.set(entity.id, entity);
    return entity;
  }

  findById(id: number): T | undefined {
    return this.store.get(id);
  }

  findAll(): T[] {
    return [...this.store.values()];
  }

  delete(id: number): boolean {
    return this.store.delete(id);
  }
}

// --- エンティティの定義 ---
interface Product extends Entity {
  name: string;
  price: number;
  stock: number;
}

interface Supplier extends Entity {
  company: string;
  email: string;
}

// 同じ Repository クラスを Product / Supplier 両方に使い回せる
const productRepo = new Repository<Product>();
const supplierRepo = new Repository<Supplier>();

productRepo.save({ id: 1, name: "商品A", price: 1000, stock: 50 });
const product = productRepo.findById(1);
console.log(product?.price);  // OK: number 型
console.log(product?.email);  // コンパイルエラー（Product に email はない）

6. ジェネリックインターフェース

インターフェースにもジェネリクスを付けることで、汎用的な契約（型の形）を定義できます。

interfaces.ts
// ページネーション付きレスポンス
interface PaginatedResult<T> {
  items: T[];
  total: number;
  page: number;
  perPage: number;
}

// 非同期のサービス層インターフェース
interface CrudService<T, CreateDto, UpdateDto> {
  findAll(page: number): Promise<PaginatedResult<T>>;
  findById(id: number): Promise<T | null>;
  create(dto: CreateDto): Promise<T>;
  update(id: number, dto: UpdateDto): Promise<T>;
  delete(id: number): Promise<void>;
}

// --- 具体的なサービスに適用 ---
type Product = { id: number; name: string; price: number };
type CreateProductDto = Omit<Product, "id">;
type UpdateProductDto = Partial<CreateProductDto>;

class ProductService implements CrudService<Product, CreateProductDto, UpdateProductDto> {
  async findAll(page: number) { /* 実装 */ return {} as any; }
  async findById(id: number) { /* 実装 */ return null; }
  async create(dto: CreateProductDto) { /* 実装 */ return {} as any; }
  async update(id: number, dto: UpdateProductDto) { /* 実装 */ return {} as any; }
  async delete(id: number) { /* 実装 */ }
}

7. 型制約（extends）

型パラメータに extends を付けることで、「T はこの型を満たすもの限定」という制約をかけられます。 制約があると、T のプロパティを安全にアクセスできます。

問題：制約なしでは T のプロパティにアクセスできない

制約なし → エラー
function printName<T>(obj: T): void {
  console.log(obj.name); // エラー: T に name があるか不明
}

解決：extends で制約を付ける

制約あり → OK
interface HasName {
  name: string;
}

// T は HasName を満たす型でないといけない
function printName<T extends HasName>(obj: T): void {
  console.log(obj.name); // OK: T は必ず name を持つ
}

printName({ name: "Alice", age: 30 }); // OK
printName({ id: 1 });                  // エラー: name がない

実例：オブジェクトの特定キーの値を取得

keyof による制約
// K は T のキーでないといけない
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const product = { id: 1, name: "商品A", price: 1000, stock: 50 };

getProperty(product, "name");   // OK: 戻り値は string
getProperty(product, "price");  // OK: 戻り値は number
getProperty(product, "email");  // エラー: "email" は Product のキーではない

実例：ソート可能な配列

sortBy.ts
function sortBy<T, K extends keyof T>(arr: T[], key: K): T[] {
  return [...arr].sort((a, b) => {
    if (a[key] < b[key]) return -1;
    if (a[key] > b[key]) return 1;
    return 0;
  });
}

const products = [
  { id: 3, name: "C商品", price: 300 },
  { id: 1, name: "A商品", price: 100 },
  { id: 2, name: "B商品", price: 200 },
];

sortBy(products, "price");  // 価格順
sortBy(products, "name");   // 名前順
sortBy(products, "stock");  // エラー: products に stock はない

8. 複数の型パラメータ

型パラメータはカンマ区切りで複数定義できます。

multiple-params.ts
// 2つの値をペアにする
function pair<A, B>(first: A, second: B): [A, B] {
  return [first, second];
}

pair("Alice", 30);     // [string, number]
pair(true, [1, 2, 3]); // [boolean, number[]]

// -----------------------------------

// キーと値の型を分けて管理するキャッシュクラス
class Cache<K, V> {
  private map = new Map<K, V>();

  set(key: K, value: V): void {
    this.map.set(key, value);
  }

  get(key: K): V | undefined {
    return this.map.get(key);
  }
}

const userCache = new Cache<number, User>();
userCache.set(1, { id: 1, name: "Alice" });
const user = userCache.get(1); // 型: User | undefined

// -----------------------------------

// 変換関数（入力と出力の型が異なる）
function transform<Input, Output>(
  items: Input[],
  fn: (item: Input) => Output
): Output[] {
  return items.map(fn);
}

const names = transform(
  [{ id: 1, name: "Alice" }, { id: 2, name: "Bob" }],
  user => user.name
);
// names: string[]  ← User[] を string[] に変換

9. 実務で使われるパターン

パターン1：Result 型（エラーハンドリング）

例外を投げずに成功・失敗を型で表現する「Result パターン」はジェネリクスの定番活用です。

result.ts
type Result<T, E = Error> =
  | { ok: true;  value: T }
  | { ok: false; error: E };

async function safeParseProduct(id: number): Promise<Result<Product>> {
  try {
    const res = await fetch(`/api/products/${id}`);
    if (!res.ok) return { ok: false, error: new Error("Not Found") };
    const data: Product = await res.json();
    return { ok: true, value: data };
  } catch (e) {
    return { ok: false, error: e as Error };
  }
}

// --- 使い方 ---
const result = await safeParseProduct(1);
if (result.ok) {
  console.log(result.value.name); // Product 型として確定
} else {
  console.error(result.error.message); // Error 型として確定
}

パターン2：フォームバリデーション

validation.ts
type ValidationResult<T> = {
  isValid: boolean;
  data?: T;
  errors: Partial<Record<keyof T, string>>;
};

function validate<T>(
  input: Partial<T>,
  rules: { [K in keyof T]?: (v: unknown) => string | null }
): ValidationResult<T> {
  const errors: any = {};
  for (const key in rules) {
    const error = rules[key]?.(input[key]);
    if (error) errors[key] = error;
  }
  const isValid = Object.keys(errors).length === 0;
  return { isValid, data: isValid ? input as T : undefined, errors };
}

// --- 使い方 ---
type CreateProductForm = { name: string; price: number };

const result = validate<CreateProductForm>(
  { name: "", price: -100 },
  {
    name:  v => !v ? "名前は必須です" : null,
    price: v => (typeof v === "number" && v < 0) ? "価格は0以上" : null,
  }
);

パターン3：イベントエミッター

event-emitter.ts
class TypedEventEmitter<Events extends Record<string, unknown>> {
  private listeners = new Map<string, Function[]>();

  on<K extends keyof Events>(event: K, fn: (data: Events[K]) => void): void {
    const existing = this.listeners.get(event as string) ?? [];
    this.listeners.set(event as string, [...existing, fn]);
  }

  emit<K extends keyof Events>(event: K, data: Events[K]): void {
    this.listeners.get(event as string)?.forEach(fn => fn(data));
  }
}

// --- 使い方（在庫管理のイベント）---
type InventoryEvents = {
  "stock:updated":  { productId: number; newStock: number };
  "order:created":  { orderId: number; items: number[] };
  "alert:low-stock": { productId: number; remaining: number };
};

const emitter = new TypedEventEmitter<InventoryEvents>();

emitter.on("stock:updated", ({ productId, newStock }) => {
  console.log(`商品 ${productId}: 在庫 ${newStock} 個に更新`);
});

emitter.emit("stock:updated", { productId: 1, newStock: 42 }); // OK
emitter.emit("stock:updated", { productId: 1, quantity: 42 }); // エラー

10. まとめ

構文書き方用途
ジェネリック関数 function f<T>(x: T): T 汎用ユーティリティ・API ラッパー
ジェネリッククラス class C<T> { ... } データ構造・リポジトリパターン
ジェネリック型/I/F interface I<T> { data: T } API レスポンス型・サービス契約
型制約 T extends HasId プロパティアクセスを安全に行う
keyof 制約 K extends keyof T 動的なキーアクセス・ソート・フィルタ
型の重複に気づく
→
型パラメータ T で抽象化
→
必要なら extends で制約
→
型安全なコードの再利用
🎯
学習の次のステップ：
TypeScript の組み込みジェネリック型（Partial<T>, Required<T>, Pick<T, K>, Omit<T, K>, ReturnType<F>）を使いこなせると、より表現力の高い型が書けます。
⚠️
やりすぎ注意： ジェネリクスは強力ですが、型パラメータが 3〜4 個になると読みにくくなります。 実際のコードでは「any を避けるため」だけに使うのではなく、 「同じロジックを複数の型で再利用する必要があるとき」に使うのが適切な判断です。
