Vite + React + TypeScript

Day4 導入ガイド

Vite でプロジェクトを作り、React の基本概念をコードで理解する。

1. Vite とは

Vite（ヴィート）は フロントエンド開発ツール です。「React アプリをブラウザで動かすための環境一式」をコマンド1つで用意してくれます。

PHP / Laravel に例えると、composer create-project laravel/laravel がプロジェクト雛形を作るのと同じ役割です。

.tsx ファイル
→
Vite（変換・バンドル）
→
ブラウザで実行

Vite が解決する問題

課題 Vite がやること
TypeScript はブラウザで直接動かない TSX → JS へ変換してブラウザに渡す
ファイルを変更するたびに手動でビルドしたくない 保存するたびにブラウザが自動更新（HMR）
本番用に複数ファイルをまとめたい npm run build で最適化済みの JS を出力
React + TypeScript の設定が複雑 テンプレートで最初から全設定済み

プロジェクト作成コマンド

Terminal
# React + TypeScript テンプレートでプロジェクト作成
npm create vite@latest day4-react -- --template react-ts

# ディレクトリへ移動して依存パッケージをインストール
cd day4-react
npm install

# 開発サーバー起動 → http://localhost:5173 で確認
npm run dev

作成されるディレクトリ構造

day4-react/
  src/
    App.tsx                ← アプリのルートコンポーネント（ここから書き始める）
    main.tsx               ← React を HTML に差し込む起点（基本触らない）
    index.css              ← グローバル CSS
    components/            ← 自作コンポーネントはここに作る
  index.html               ← エントリーポイント HTML（<div id="root"> がある）
  tsconfig.json           ← TypeScript 設定（自動生成済み）
  vite.config.ts         ← Vite 設定（基本触らない）
  package.json            ← npm scripts が定義されている
💡
触るのは src/ の中だけで大丈夫です。main.tsx・vite.config.ts・tsconfig.json は Day4 では変更不要です。

2. React とは

React は UI をコンポーネント単位で組み立てる JavaScript ライブラリです。Meta（旧 Facebook）が開発・OSS 公開しています。

PHP / Laravel の Blade テンプレートに似ていますが、React は「状態が変わったときに自動で画面を更新する」仕組みを持っています。

コンポーネント指向とは

画面を「部品（コンポーネント）」に分割して組み合わせます。

App（ページ全体）
<App />
商品リスト
<ItemList />
検索フォーム
<ItemSearch />
在庫カード
<ItemCard />
✅
各コンポーネントは 独立したファイル（.tsx）として作成し、App.tsx から呼び出します。Laravel でいうと Blade コンポーネント（@component）に近いイメージです。

3. JSX / TSX — HTML を JS の中に書く構文

📌
JSX（JavaScript XML）は HTML に似た構文を JavaScript の中に書けるようにした記法です。
TSX は JSX を TypeScript で書いたもの（.tsx 拡張子）です。
Vite が変換してくれるので、ブラウザに送る前に通常の JavaScript になります。
PHP / Blade（参考）
<div class="card">
  <p>{{ $item->name }}</p>
  <p>¥{{ $item->price }}</p>
</div>
TSX（React）
const name = "ペン";
const price = 120;

// {} の中は JavaScript 式を書ける
return (
  <div className="card">
    <p>{name}</p>
    <p>¥{price}</p>
  </div>
);

JSX の注意点

HTMLJSX / TSX理由
class="..." className="..." class は JS の予約語のため
for="..." htmlFor="..." for は JS の予約語のため
タグを閉じなくてよい
<br>		 必ず閉じる
<br />		 JSX は XML の厳密な構文ルール
複数タグを並べてよい 1つのルート要素で包む
<>...</>（Fragment）		 JSX は1つの要素を返す必要がある

4. 関数コンポーネント

React のコンポーネントは JSX を返す TypeScript 関数です。関数名は大文字始まりにします。

src/components/ItemCard.tsx
// Props の型を interface で定義する
interface ItemCardProps {
  name: string;
  price: number;
}

// 関数コンポーネント: Props を受け取り JSX を返す
export function ItemCard({ name, price }: ItemCardProps) {
  return (
    <div className="card">
      <p>{name}</p>
      <p>¥{price}</p>
    </div>
  );
}
src/App.tsx（呼び出し側）
import { ItemCard } from "./components/ItemCard";

export function App() {
  return (
    <div>
      {/* コンポーネントを HTML タグのように使う */}
      <ItemCard name="ペン" price={120} />
      <ItemCard name="ノート" price={150} />
    </div>
  );
}
💡
Props は「親コンポーネントから子コンポーネントへデータを渡す仕組み」です。
Laravel に例えると、Blade コンポーネントに :name="$item->name" で値を渡すのと同じです。
Props は子から親へは変更できません（一方通行）。

Optional Props（省略可能な Props）

? を付けると省略可能になる
interface ItemCardProps {
  name: string;
  price: number;
  category?: string;  // ? で省略可能（string | undefined になる）
}

export function ItemCard({ name, price, category }: ItemCardProps) {
  return (
    <div>
      <p>{name} — ¥{price}</p>
      {/* category が undefined のときは何も表示しない */}
      {category && <p>{category}</p>}
    </div>
  );
}

// 両方コンパイルエラーにならない
<ItemCard name="ペン" price={120} category="文具" />
<ItemCard name="ノート" price={150} />

5. useState — コンポーネントの状態管理

📌
useState は React が提供する Hook（フック）の一つです。
コンポーネントの中に「変化する値（状態）」を持たせ、値が変わると自動で画面を再レンダリングします。
PHP / Laravel の変数とは異なり、値を直接変更しても画面は更新されません。必ず専用の関数（setter）を使います。
NG: 直接変更しても画面は更新されない
let qty = 10;

function handleClick() {
  qty = qty + 1;  // 値は変わるが
  // 画面は更新されない！
}
OK: useState の setter を使う
import { useState } from "react";

const [qty, setQty] = useState(10);
//    ↑現在値  ↑更新関数  ↑初期値

function handleClick() {
  setQty(qty + 1); // 画面が自動更新される
}

在庫カウンターの実装例

src/components/StockCounter.tsx
import { useState } from "react";

interface StockCounterProps {
  initialQty: number;
}

export function StockCounter({ initialQty }: StockCounterProps) {
  const [qty, setQty] = useState(initialQty);

  return (
    <div>
      <p>在庫数: {qty}</p>
      <button onClick={() => setQty(qty + 1)}>入庫 +1</button>
      <button onClick={() => setQty(Math.max(0, qty - 1))}>出庫 -1</button>
    </div>
  );
}

テキスト入力の状態管理（制御コンポーネント）

入力値を state で管理する
const [query, setQuery] = useState("");

return (
  <div>
    <input
      value={query}
      onChange={(e) => setQuery(e.target.value)}
      placeholder="商品名を入力"
    />
    {query && <p>検索中: {query}</p>}
  </div>
);
⚠️
useState などの Hook は必ずコンポーネントのトップレベルで呼び出すルールがあります。if や for の中で呼ぶとエラーになります。
また、Hook を使うコンポーネントのファイル先頭に "use client" は不要です（Vite 環境では関係ありません。Next.js に移行したときに登場します）。

6. リスト表示 / 条件レンダリング

リスト表示：.map() で繰り返す

配列を .map() で JSX に変換する
interface Item {
  id: number;
  name: string;
  qty: number;
}

const items: Item[] = [
  { id: 1, name: "ペン",   qty: 50 },
  { id: 2, name: "ノート", qty: 8  },
];

return (
  <ul>
    {items.map((item) => (
      {/* key は React が差分更新に使う識別子。id など一意の値を渡す */}
      <li key={item.id}>
        {item.name} — {item.qty}個
      </li>
    ))}
  </ul>
);
⚠️
key を省略すると React が警告を出します。配列インデックス（index）を key にすることは可能ですが、リストの並び替えがあると不具合の原因になります。基本は id など一意の値を使います。

条件レンダリング

3つのパターン
// ① && 演算子: 条件が true のときだけ表示
{qty <= 10 && <span>⚠ 在庫少</span>}

// ② 三項演算子: true/false で表示内容を切り替え
{inStock ? <span>在庫あり</span> : <span>在庫なし</span>}

// ③ 早期 return: 空の場合は別の JSX を返す
if (items.length === 0) {
  return <p>商品が登録されていません</p>;
}
return <ul>{items.map(...)}</ul>;

7. React のデータフロー（全体像）

React のデータは常に親 → 子の一方向に流れます。

App（親）
state を持つ
→ Props でデータを渡す
ItemList（子）
items を受け取る
→ さらに Props を渡す
ItemCard（孫）
name, qty を受け取る

子から親へのデータ伝達が必要なときは「コールバック関数を Props で渡す」パターンを使います（Day5 以降で扱います）。
