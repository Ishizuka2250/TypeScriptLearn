# スキルギャップ診断結果（2026-05-18）

## 目的
TypeScript / Next.js 学習開始前に、自己認識と実力のズレを確認し、4週間プランを実行可能な難易度へ調整する。

## 回答サマリ
- JS自己評価: 3（基礎〜実務の中間）
- `var` / `let` / `const`: 再代入は理解、スコープ/巻き上げは未整理
- Promise系: `Promise.all` と `allSettled` の使い分けは未経験
- イベントループ問題: 回答は `1,4,2,3`（正答は `1,4,3,2`）
- 配列メソッド: `map` / `filter` は概ね理解、`reduce` は不明瞭
- エラー処理: ネットワーク失敗とHTTPエラーの区別は良好
- イベント委譲: 概念は部分理解（DOMイベントの親要素集約観点を補強したい）
- TypeScript: 実務経験なし
- React: 実務経験なし
- Next.js: 実務経験なし

## 診断結果（現時点）

### JavaScript
- 強み
  - APIエラー分類の理解が実務的
  - `map` / `filter` の用途理解あり
- 強化優先
  - スコープ/巻き上げ（`var` と `let`/`const` の差）
  - 非同期制御（Microtask / Macrotask、Promise合成）
  - `reduce` とデータ集計
  - イベント委譲の実装パターン
- 判定
  - 基礎中級手前（業務で使える領域あり、ただし設計寄りの理解は要補強）

### TypeScript
- 判定
  - 初級（これから開始）
- 最短到達の鍵
  - `strict` 前提で、ドメイン型を先に固定する

### React / Next.js
- 判定
  - 初級（未経験）
- 最短到達の鍵
  - React基礎を短期で固め、Next.js App Routerに早期接続する

## 認識差の結論
「JavaScriptは中級」の認識に対し、実態は「基礎機能は使えるが、非同期/スコープ/設計の根拠が弱い」状態。案件でのズレが出やすいのは、主に以下。
- 非同期処理の挙動見誤り
- TypeScript導入時の型設計方針不足
- React/Next.js未経験による初速低下

## 対応方針（4週間）
- Week 1前半をJavaScript穴埋めに再配分
- Week 1後半からTypeScriptに接続
- Week 2でReact最小セット + Next.js画面
- Week 3でAPI/DB
- Week 4でテスト/仕上げ
